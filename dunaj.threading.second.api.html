dunaj.threading.second API

Copyright © 2013, 2015,  Jozef Wagner
version 0.3.7
Additional copyright for parts of documentation and/or underlying implementation: Copyright © 2013, 2015, LonoCloud.

Macros for syntax threading into second item in forms.

See dunaj.threading.last for macros which thread into last item instead.

Using synthread for underlying implementation.
The diamond symbol <> is bound and rebound to the current topic as it is threaded through the forms.

Idiomatic use is to alias this namespace under -> symbol.

Documentation needs more work.

(ns foo.bar
  (:api dunaj)
  (:require [dunaj.threading.second :as ->]))

(-> {:a 1 :b 2}
    (->/update :b inc)
    (->/when (< (rand) 0.5)
      (->/update :a dec)))
;;=> {:b 3, :a 1}
;; or
;;=> {:b 3, :a 0}

(-> [{:foo1 0 :foo2 1 :foo3 -10} "a-bar"]
    (->/first
     (do (reduce + (map val <>))))
    (->/second keyword))
;;=> [-9 :a-bar]

cond

Available since version 1.0 (alias of lonocloud.synthread/cond)

not referred automatically

MACRO (cond x & test-form-pairs)

EXPERIMENTAL Thread x through forms in each clause. Return x if no test matches. (→/cond [1 2] true (conj 3) false pop)

do

Available since version 1.0 (alias of lonocloud.synthread/do)

not referred automatically

MACRO (do x & body)

Thread x through body. Semantically identical to → with the extra feature that the symbol <> is bound to the new value of x between each form.

Note that the old marking constraint has been removed as it was unduly restrictive.

fn

Available since version 1.0 (alias of lonocloud.synthread/fn)

not referred automatically

MACRO (fn args & body)

Thread x into body of fn. (inspired by Prismatic’s fn→). (let [add-n (→/fn [n] (+ n))] (→ 1 (add-n 2))) ;; returns 3

for

Available since version 1.0 (alias of lonocloud.synthread/for)

not referred automatically

MACRO (for x seq-exprs & body)

Thread x through each iteration of body. Uses standard looping binding syntax for iterating. (→/for 4 [x [1 2 3]] (+ x)) ;; returns 10

if

Available since version 1.0 (alias of lonocloud.synthread/if)

not referred automatically

MACRO (if x pred then else)

If pred is true, thread x through the then form, otherwise through the else form. (→ 5 (→/if should-inc? inc dec))

if-let

Available since version 1.0 (alias of lonocloud.synthread/if-let)

not referred automatically

MACRO (if-let x [local pred :as binding] then else)

Thread x through then or else depending on the value of pred. If pred is true, bind local to pred. (→ {} (→/if-let [x :bar] (assoc :foo x) (assoc :was-bar false))) ;; returns {:foo :bar}

update

Available since version 1.0 (alias of lonocloud.synthread/assoc)

not referred automatically

MACRO (update x & key-form-pairs)

Thread the value at each key through the pair form. The form must be a function which accepts the value of the key as its first argument, e.g.

=> (-> {:a 1}
     (->/assoc :a inc))
{:a 2}

This macro DOES NOT work like clojure.core/assoc. If you just want to set a key to some value, use clojure.core/assoc, e.g.

=> (-> {:a 1}
     (assoc :b 2))
{:a 1, :b 2}

Available since version 1.0 (alias of lonocloud.synthread/in)

not referred automatically

MACRO (update-in x path & body)

Thread the portion of x specified by path through body. (→/in {:a 1, :b 2} [:a] (+ 2)) ;; = {:a 3, :b 2}

Available since version 1.0 (alias of lonocloud.synthread/when)

MACRO (when x pred & body)

If pred is true, thread x through body, otherwise return x unchanged. (→ 5 (→/when should-inc? inc))

when-let

Available since version 1.0 (alias of lonocloud.synthread/when-let)

not referred automatically

MACRO (when-let x [local pred :as binding] & forms)

If bound values are true in bindings, thread x through the body, otherwise return x unchanged. (→ 5 (→/when-let [amount (:amount foo)] (+ amount)))
