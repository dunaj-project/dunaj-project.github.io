dunaj.math.precise API

Copyright © 2013, 2015,  Jozef Wagner
version 0.4.0
Additional copyright for parts of documentation and/or underlying implementation: Copyright © 2008, 2015, Rich Hickey and Clojure contributors.

Precise math with an arbitrary precision.

Slower than normal math operations, but auto-promotes when needed.

Depending on the needs, there are several other namespaces with similar functionalities:

  • dunaj.math which is slightly faster but subject to rounding and truncation

  • dunaj.math.unchecked, much faster but subject to overflow

  • dunaj.host.int, host specific unchecked operations with fastest performance.

*

Available since version 1.0 (alias of clojure.core/*')

not referred automatically

Usage:

  • (*)

  • (* x)

  • (* x y)

  • (* x y & more)

Type signatures:

  • () → Number

  • (Number) → Number

  • (Number ⨯ Number) → Number

  • (Number ⨯ Number ⨯ (Va Number)) → Number

Returns the product of nums. (*) returns 1. Supports arbitrary precision.

See also: multiply, dunaj.math/, dunaj.math.unchecked/, dunaj.host.int/imul

+

Available since version 1.0 (alias of clojure.core/+')

not referred automatically

Usage:

  • (+)

  • (+ x)

  • (+ x y)

  • (+ x y & more)

Type signatures:

  • () → Number

  • (Number) → Number

  • (Number ⨯ Number) → Number

  • (Number ⨯ Number ⨯ (Va Number)) → Number

Returns the sum of nums. (+) returns 0. Supports arbitrary precision.

See also: add, inc, dunaj.math/+, dunaj.math.unchecked/+, dunaj.host.int/iadd

-

Available since version 1.0 (alias of clojure.core/-')

not referred automatically

Usage:

  • (- x)

  • (- x y)

  • (- x y & more)

Type signatures:

  • (Number) → Number

  • (Number ⨯ Number) → Number

  • (Number ⨯ Number ⨯ (Va Number)) → Number

If no ys are supplied, returns the negation of x, else subtracts the ys from x and returns the result. Supports arbitrary precision.

See also: negate, subtract, dec, dunaj.math/-, dunaj.math.unchecked/-, dunaj.host.int/isub
